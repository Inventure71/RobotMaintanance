2. Terminal Context

Capture the command outputs that make the test objectively evaluable.

Activate the terminal, run Run generic info commands first, then add the robot-specific commands you normally use to inspect the robot or stack. The goal is to capture outputs that let the generated checks evaluate the system correctly.

The transcript mainly helps the model infer read and validation blocks. It should not be expected to invent the correct commands if the transcript does not show them.

The generic-info action is meant to capture baseline context such as OS, ROS, processes, networking, services, and containers before you add any topic or application-specific checks.

ROS1

rostopic list rosnode list rostopic echo /topic -n 1

ROS2

ros2 topic list ros2 node list ros2 topic echo /topic --once

Linux / System

systemctl status ... journalctl -u ... -n 100 ip addr ps aux

Containers

docker ps docker logs <container> --tail 100

3. Ask Your LLM

Describe the stack and the exact test objective. Make sure the transcript already includes the generic-info run plus any robot-specific commands you need. The prompt bundle will be generated automatically when you continue.

4. Copy Prompt And Paste Result

Copy the generated prompt into your external LLM, then paste the returned JSON here.

Click inside the prompt box, press Ctrl/Cmd+A, then Ctrl/Cmd+C. Paste that exact prompt into your external LLM before returning the JSON below.

Paste the prompt into your external LLM provider, ask it to return only JSON, then paste the result below and validate it. The current draft stays unchanged until validation succeeds.

What is a JSON?

Use this shape to identify valid JSON output:

{
  "id": "battery_health_check",
  "label": "Battery Health Check",
  "mode": "orchestrate",
  "execute": [
    {
      "id": "step_1",
      "command": "ros2 topic list",
      "saveAs": "topics_raw"
    }
  ],
  "checks": [
    {
      "id": "battery_health_check__topics",
      "label": "Has battery topic",
      "runAtConnection": true,
      "read": {
        "kind": "contains_string",
        "inputRef": "topics_raw",
        "needle": "/battery",
        "caseSensitive": false
      },
      "pass": {
        "status": "ok",
        "value": "present",
        "details": "The required battery topic is present."
      },
      "fail": {
        "status": "error",
        "value": "missing",
        "details": "The required battery topic is missing."
      }
    }
  ]
}
